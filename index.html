ما هي الحوسبة السحابية؟

الحوسبة السحابية هي تقديم خدمات الحوسبة عبر الإنترنت — مثل الخوادم والتخزين وقواعد البيانات والشبكات والبرمجيات — بدلًا من تشغيلها محليًا على جهازك.

لماذا نتعلمها؟

  • قابلية التوسع
  • إطلاق التطبيقات بسرعة

نظرة عامة

تتوزع خدمات الحوسبة السحابية بشكل عام إلى ثلاث طبقات رئيسية:

IaaS — البنية التحتية كخدمة

خوادم افتراضية، تخزين، شبكات. أنت تدير النظام التشغيلي والتطبيقات.

PaaS — النظام كمنصة

منصة لبناء تطبيقات دون إدارة البنية التحتية (قاعدة بيانات مُدارة، runtime).

SaaS — البرمجيات كخدمة

تطبيق جاهز يعمل عبر الإنترنت (مثل البريد الإلكتروني كخدمة أو أدوات الإنتاجية).

أمثلة على خدمات سحابية مشهورة

  • AWS — Amazon Web Services
  • Microsoft Azure
  • Google Cloud Platform
  • خدمات خاصة (Private Cloud)

درس سريع — رفع موقع بسيط إلى سحابة (مفاهيم)

  1. أنشئ حسابًا على مزود سحابي.
  2. أنشئ خادم افتراضي أو خدمة استضافة.
  3. حمّل ملفات الموقع وأشر إلى اسم النطاق.
  4. ضبط صلاحيات الأمان (قواعد الجدار الناري).

مثال سريع على ملف HTML يمكن رفعه:

<!doctype html>
<html lang="ar">
<head><meta charset="utf-8"><title>مثال</title></head>
<body>مرحبا سحابة</body>
</html>

نصيحة: ابدأ بخدمة استضافة مجانية أو خطة مجانية للمزود لتجربة بيئة العمل.

الدرس الأول — أوامر Git الأساسية

في هذا الدرس ستتعرف على أهم أوامر Git التي تحتاجها لإدارة مشاريعك وحفظ الإصدارات.

تهيئة المستودع

git init

إضافة الملفات للتتبع

git add .
git add filename.txt

حفظ التغييرات

git commit -m "رسالة التغيير"

عرض حالة المشروع

git status

عرض تاريخ الإصدارات

git log

إنشاء فرع جديد

git branch new-branch

الانتقال بين الفروع

git checkout new-branch

دمج الفروع

git merge new-branch

ربط المشروع بمستودع GitHub

git remote add origin https://github.com/username/repo.git

رفع الملفات إلى GitHub

git push -u origin main

الدرس الثاني — عملية التعاون Collaboration

في هذا الدرس ستتعلم كيف يعمل المطورون معًا على نفس المشروع باستخدام Git و GitHub.

1) استنساخ المشروع (Clone)

يستخدم لاستنساخ المستودع من GitHub إلى جهازك:

git clone https://github.com/username/repo.git

2) إضافة فرع جديد للعمل عليه

git branch feature-login
git checkout feature-login

3) عمل تعديلات ثم رفعها

git add .
git commit -m "إضافة ميزة تسجيل الدخول"
git push origin feature-login

4) إنشاء Pull Request

يتم من خلال GitHub لطلب دمج فرعك مع الفرع الرئيسي.

5) مراجعة الكود (Code Review)

يقوم الفريق بقراءة الأكواد وتقديم ملاحظات قبل الدمج.

6) حل التعارضات (Merge Conflicts)

git pull origin main
# تعديل الملفات المتعارضة
git add .
git commit -m "حل التعارضات"

7) دمج الفرع مع الفرع الرئيسي

git checkout main
git merge feature-login

8) جلب آخر تحديثات الفريق

git pull origin main
